## Как работать в течение недели

**1. Найди свой TODO-файл** в папке текущей недели (`tasks/week-XX/`) — там одна задача, разбитая на 3 шага.

**2. Создай ветку** от `main`:
```bash
git checkout main
git pull
git checkout -b feature/<твоя-ветка>
```

**3. Делай шаги по очереди.** После каждого шага — коммит:
```bash
git add .
git commit -m "feat: ..."
```
Итого: 3 шага = 3 коммита за неделю.

**4. Когда всё готово — открой Pull Request в `main`:**
- Название: коротко что сделано
- Описание: что именно сделал и зачем
- Назначь кого-нибудь из команды на ревью

**5. Проревьюй чужой PR.** Каждый делает минимум одно ревью в неделю.

## Правила в двух словах

- В `main` напрямую не пушим — только через PR
- Хардкода нет — ключи и конфиги в `.env`
- `.env` не коммитится — только `.env.example`
- Комментарии в коде — по-русски
- Непонятно как сделать — смотри [ARCHITECTURE.md](ARCHITECTURE.md), спрашивай в чате
