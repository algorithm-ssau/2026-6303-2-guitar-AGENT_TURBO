# PRD: AI-агент по подбору гитар (с интеграцией поиска на Reverb)

## 1. Цель продукта

Создать чат-агента, который:

1. Принимает абстрактные предпочтения пользователя по звуку, стилю и ощущениям.
2. Переводит их в технические параметры инструмента.
3. Вызывает существующий скрипт поиска на Reverb.
4. Возвращает пользователю только релевантные ссылки на инструменты.

Доставка, регион, оформление заказа — вне зоны ответственности продукта.

---

## 2. Область продукта (Scope)

### Включено

* Подбор гитар (электро, акустика, классика — в зависимости от запроса).
* Интерпретация абстрактных характеристик (“тёплый”, “яркий”, “плотный”, “для джаза” и т. п.).
* Формирование параметров поиска.
* Вызов скрипта поиска Reverb.
* Ранжирование результатов.
* Выдача ссылок.
* Тематические консультации (звук, датчики, дерево, мензура и т. п.).

### Исключено

* Оформление заказа.
* Проверка продавцов.
* Гарантии актуальности цены.
* Доставка и логистика.
* Интеграция с оплатой.

---

## 3. Целевая аудитория

* Начинающие гитаристы.
* Музыканты, ищущие конкретный звук.
* Пользователи, не разбирающиеся в технических характеристиках.
* Люди, которые формулируют запрос в абстрактных терминах.

---

## 4. Пользовательские сценарии (User Flows)

### Сценарий 1: Абстрактный запрос

**Ввод:**
“Хочу тёплый джазовый звук, мягкая атака, чтобы не звенела, бюджет до 1000.”

**Процесс:**

1. Агент извлекает:

   * тип инструмента (если не указан — определяет по контексту или уточняет),
   * бюджет,
   * тональные характеристики,
   * жанр.
2. Преобразует в технические параметры.
3. Вызывает скрипт поиска.
4. Возвращает список ссылок (Top N).

**Вывод:**
Список ссылок на подходящие инструменты.

---

### Сценарий 2: Конкретный запрос

**Ввод:**
“Telecaster до 800 долларов.”

**Процесс:**

* Минимальная обработка.
* Прямой поиск.
* Возврат ссылок.

---

### Сценарий 3: Консультация без поиска

**Ввод:**
“Чем отличаются P90 от хамбакеров?”

**Процесс:**
Ответ без вызова поиска.

---

## 5. Функциональные требования

### 5.1 Обработка запроса

Агент должен:

* Определять тип инструмента.
* Выделять бюджет.
* Выделять стиль/жанр.
* Интерпретировать тональные признаки.
* Извлекать ограничения.

### 5.2 Модуль интерпретации звука

Создаётся внутренняя система маппинга:

| Абстракция   | Возможные технические параметры   |
| ------------ | --------------------------------- |
| Тёплый       | HH, P90, neck pickup, semi-hollow |
| Яркий        | Single coil, 25.5 scale           |
| Плотный      | HH high output                    |
| Мягкая атака | Hollow/semi-hollow, lower output  |
| Металл       | HH, high output, 24 лада          |
| Джаз         | Hollow/semi-hollow, neck pickup   |

Маппинг реализуется в системном промпте агента.

---

### 5.3 Интеграция с поисковым скриптом

Агент формирует нормализованный JSON с параметрами и вызывает:

```
search_reverb(params)
```

Скрипт возвращает список объявлений.

Агент:

* Не изменяет ссылки.
* Не выдумывает характеристики.
* Не модифицирует цену.
* Использует только реальные данные из ответа скрипта.

---

### 5.4 Ранжирование

Критерии:

* Соответствие типу инструмента.
* Совпадение по датчикам.
* Совпадение по ключевым словам.
* Попадание в бюджет.
* Релевантность стилю.

Результат: Top 3–5 ссылок.

---

### 5.5 Формат ответа пользователю

Пользователь получает:

* Краткое пояснение (1–2 строки).
* Список ссылок.

Пример:

```
Подходящие варианты:

1. [Название инструмента]
https://reverb.com/...

2. ...
```

Без обсуждения доставки и региона.

---

## 6. Нефункциональные требования

* Время ответа: < 10 секунд.
* Не более 5 ссылок за выдачу.
* Отсутствие вымышленных данных.
* Поддержка русского языка (приоритет).
* Масштабируемость под другие маркетплейсы.

---

## 7. Архитектура (верхний уровень)

### Компоненты

1. **Chat Interface**
2. **LLM-агент**

   * Интерпретация запроса
   * Генерация search params
3. **Reverb Search Script**
4. **Ranking Layer**
5. **Response Formatter**

### Поток данных

User → LLM → JSON params → Reverb script → Results → Ranking → Links → User

---

## 8. Ограничения

* Регион не учитывается.
* Актуальность цены зависит от Reverb.
* Результаты ограничены возможностями скрипта.

---

## 9. MVP

Минимальная версия включает:

* Электро- и акустические гитары.
* Бюджетный фильтр.
* Маппинг 15–20 ключевых абстрактных характеристик.
* Выдачу 3–5 ссылок.
* Базовый скоринг.

Без:

* Истории пользователя.
* Персонализации.
* Кэширования.
* Поддержки других маркетплейсов.

---

## 10. Будущие расширения

* Поддержка Reverb + eBay.
* Учет состояния инструмента.
* Сохранение профиля пользователя.
* Обучение на пользовательском фидбеке.
* Подбор не только гитар, но и усилителей, педалей.

---

## 11. Критерии успеха

* Пользователь получает релевантные инструменты без уточняющих вопросов в ≥70% случаев.
* Минимальное количество нерелевантных ссылок.
* Повторные обращения пользователя.
* Среднее число сообщений до выдачи ссылок ≤ 3.
