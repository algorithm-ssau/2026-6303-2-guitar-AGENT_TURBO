# Исследование интеграции с Reverb

Дата исследования: 15 марта 2026
Ответственный: Сидоров Артемий
Дата пересмотра: 18 апреля 2026

## Что исследовано

Для первой итерации я посмотрел, как у Reverb устроен поиск объявлений, и проверил оба варианта получения данных программно: через официальный API и через парсинг страниц поиска. По итогам сравнения и обсуждения с командой остановились на собственном парсере страниц.

Источники:
- страницы поиска Reverb: `https://reverb.com/marketplace`
- структура карточек объявлений: HTML-разметка результатов поиска

## Почему отказались от официального Reverb API

- Доступ ко многим методам требует одобренного партнёрского аккаунта и API-токена; для учебного открытого репозитория это лишняя зависимость и риск утечки ключа.
- Часть фильтров и сортировок, которые видны на сайте, в публичном API недоступна или ограничена.
- API накладывает rate limits и условия использования, под которые сложно подстраивать учебный проект с нерегулярной нагрузкой.

## Почему выбрали собственный парсер

- Полная автономность: интеграция работает без сторонних ключей и аккаунтов.
- Покрывает ровно тот же набор фильтров, который видит обычный пользователь на сайте.
- Контроль над форматом нормализованного ответа: парсер сразу возвращает поля, нужные модулю ранжирования (`title`, `price`, `image_url`, `listing_url` и т. п.).
- Лёгко переключается на локальный набор мок-данных через `USE_MOCK_REVERB` — это сохраняет полный пайплайн при отсутствии сети.

Минусы и как мы их закрываем:
- Хрупкость к изменениям вёрстки → тесты на фиксированных HTML-фикстурах и быстрая правка селекторов.
- Антибот-ограничения → headless-браузер (Playwright/Puppeteer), human-like задержки, ограничение частоты запросов.

## Рекомендация для проекта

Реализуем интеграцию с Reverb как собственный парсер. Шаги:

1. Зафиксировать выбор в `STACK.md` (сделано).
2. Уточнить минимальный набор параметров для поиска: `search_queries`, `price_min`, `price_max`.
3. Реализовать парсер в `backend/search/search_reverb.py`, который принимает параметры поиска и возвращает нормализованный список объявлений (на текущий момент работает на мок-данных, скрэпер подключается отдельной задачей).
4. Сохранить режим работы с локальным набором данных (`USE_MOCK_REVERB`) для тестов и воспроизводимой демонстрации.

## Что ещё нужно уточнить
