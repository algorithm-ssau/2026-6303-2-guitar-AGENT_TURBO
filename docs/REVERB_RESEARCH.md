# Исследование интеграции с Reverb

Дата исследования: 15 марта 2026
Ответственный: Сидоров Артемий

## Что исследовано

Для первой итерации я посмотрел, как у Reverb устроен поиск объявлений, и проверил, есть ли официальный способ получать данные программно. В актуальной документации нашёл публичное API Reverb, в котором описаны ресурсы для работы с объявлениями, фильтрами и параметрами поиска. Это лучше подходит под наш проект, чем прямой парсинг HTML-страниц.

Источники:
- официальный API-документ Reverb: `https://api.reverb.com/api`
- JSON-описание API: `https://reverb.com/api/doc.json`

## Есть ли API

Да, у Reverb есть официальный HTTP API. По документации видно, что поиск можно строить через запросы к объявлениям с параметрами фильтрации. Для нашего MVP это означает, что backend сможет отправлять нормализованные search-параметры и получать структурированный ответ без разбора HTML.

Плюсы API-подхода:
- ответ уже структурирован и лучше подходит для Python-обработки;
- меньше риск поломки при изменении вёрстки сайта;
- проще поддерживать и тестировать;
- легче встроить обработку ошибок и ограничение количества результатов.

## Можно ли парсить сайт

Технически парсинг страниц поиска тоже возможен, но это запасной вариант. У такого подхода есть риски: HTML может меняться без предупреждения, на сайте могут быть антибот-ограничения, а сама интеграция получится более хрупкой и трудной в сопровождении. Для командного учебного проекта это не лучший базовый путь, если уже есть официальный API.

## Рекомендация для проекта

Предлагаю использовать официальный Reverb API как основной способ интеграции. На следующем этапе нужно:

1. Зафиксировать API-подход в `STACK.md`.
2. Уточнить минимальный набор параметров для поиска: `search_queries`, `price_min`, `price_max`.
3. Реализовать функцию-обёртку в `backend/search/search_reverb.py`, которая принимает параметры поиска и возвращает нормализованный список объявлений.
4. Оставить парсинг только как fallback-вариант, если позже столкнёмся с ограничениями API.

## Что ещё нужно уточнить

- Нужна ли авторизация для всех сценариев поиска или только для части методов.
- Какие именно поля из ответа Reverb нужны команде для ранжирования и вывода ссылок.
- Есть ли у API ограничения по частоте запросов, которые надо учесть перед интеграцией в MVP.
