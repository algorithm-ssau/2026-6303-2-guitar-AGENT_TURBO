# Frontend Release QA Checklist

Чеклист для ручной проверки интерфейса перед защитой. Отмечайте галочкой каждое состояние на каждом разрешении.

| Состояние | Что проверить | Desktop 1440 | Tablet 768 | Mobile 375 |
|-----------|---------------|--------------|------------|------------|
| 1. Пустой чат | Отображается welcome-экран с эмодзи 🎸, заголовком «Добро пожаловать в Guitar Agent!», описанием и примером запроса | ☐ | ☐ | ☐ |
| 2. Welcome / начальное состояние | Заголовок «REVERB AGENT» виден, placeholder в input читается полностью, кнопка «Отправить» активна | ☐ | ☐ | ☐ |
| 3. Search-ответ с карточками | Карточки гитар отображаются в гриде, каждая содержит фото (или PHOTO-плейсхолдер), title, цену, ссылку на Reverb, номер позиции и бейдж релевантности | ☐ | ☐ | ☐ |
| 4. Consultation-ответ | Текст ответа рендерится через markdown (заголовки, списки, **жирный**), виден ModeBadge «consultation», есть кнопка «Копировать» | ☐ | ☐ | ☐ |
| 5. Empty results | После search-запроса без результатов показывается блок «По вашему запросу ничего не найдено» с предложением изменить параметры | ☐ | ☐ | ☐ |
| 6. Error state | Виден блок ошибки с иконкой ⚠️, текстом ошибки и кнопкой «Попробовать снова» | ☐ | ☐ | ☐ |
| 7. Loading state | Спиннер анимируется, виден текст «Загрузка чата» и описание | ☐ | ☐ | ☐ |
| 8. Длинный ответ | Текст не вылезает за пределы бабла, вертикальный скролл работает, переносы строк корректны | ☐ | ☐ | ☐ |
| 9. Длинный title карточки | Title обрезается через ellipsis, максимум 2 строки, не ломает layout карточки | ☐ | ☐ | ☐ |
| 10. Светлая/тёмная тема | Переключение темы работает, все элементы читаемы в обеих темах, нет контрастных багов | ☐ | ☐ | ☐ |
| 11. Сайдбар | Сайдбар открывается/закрывается по кнопке ☰, список сессий скроллится, на мобильных — оверлей | ☐ | ☐ | ☐ |
| 12. Мобильный input | На 375px textarea не перекрывается клавиатурой, кнопка отправки доступна, placeholder не обрезается | ☐ | ☐ | ☐ |

## Как использовать

1. Откройте `ChatDemoScenarios` локально (импортируйте временно в `App.tsx` или откройте через Storybook/Vite preview).
2. Переключите сценарий через боковую панель.
3. Проверьте каждое состояние на трёх разрешениях (DevTools → Device Toolbar).
4. Отметьте галочкой пройденные проверки.
