УДК 004.89

ИНТЕЛЛЕКТУАЛЬНЫЙ АГЕНТ ПОДБОРА ГИТАР НА ОСНОВЕ ОБРАБОТКИ ЕСТЕСТВЕННОГО ЯЗЫКА И МНОГОКРИТЕРИАЛЬНОГО РАНЖИРОВАНИЯ

Ключевые слова: рекомендательная система, большая языковая модель, обработка естественного языка, многокритериальное ранжирование, подбор гитар

При выборе гитары на онлайн-площадках пользователи описывают желаемый инструмент субъективными терминами: «тёплый звук», «яркий тембр», «агрессивное звучание». Существующие поисковые интерфейсы маркетплейсов оперируют техническими параметрами — тип звукоснимателей, порода дерева, конфигурация электроники, — что формирует семантический разрыв между намерением пользователя и возможностями системы поиска. Задача настоящей работы — разработка интеллектуального агента, преобразующего естественно-языковые описания в структурированные поисковые запросы к маркетплейсу Reverb с последующим ранжированием результатов по степени соответствия исходному запросу.

Предлагаемое решение использует большую языковую модель Llama 3.3-70B [1, 2] в качестве ядра семантического анализа. Агент функционирует в двух режимах: поиск и консультация. Классификация режима выполняется алгоритмом на основе регулярных выражений с анализом лексических маркеров запроса (глаголы действия, ценовые паттерны, вопросительные конструкции). В режиме поиска модель извлекает структурированные параметры (тип инструмента, ценовой диапазон, звукосниматели, бренд) из свободного текста, опираясь на предопределённую таблицу маппинга абстрактных звуковых характеристик в технические спецификации. Например, характеристика «тёплый звук» транслируется в параметры: звукосниматели типа humbucker или P90, корпус из красного дерева, бренды Gibson/Epiphone. В режиме консультации модель генерирует экспертный ответ на теоретические вопросы.

Для ранжирования найденных объявлений разработан алгоритм взвешенной многокритериальной оценки [3]. Итоговый балл каждого результата вычисляется как взвешенная сумма оценок по пяти критериям: соответствие бюджету (вес 0,30), тип инструмента (0,25), конфигурация звукоснимателей (0,20), релевантность заголовка (0,15) и бренд (0,10). Каждый критерий оценивается по шкале от 0 до 100 баллов с промежуточными градациями. Алгоритм адаптивен: при неполноте входных данных формула автоматически упрощается до двух критериев — бюджет (0,55) и заголовок (0,45), — что позволяет сохранить качество ранжирования при минимальном наборе извлечённых параметров.

Система реализована в клиент-серверной архитектуре. Серверная часть построена на фреймворке FastAPI (Python) с модульной организацией: модуль агента осуществляет оркестрацию конвейера обработки и взаимодействие с LLM через Groq API; модуль поиска интегрирован с Reverb API v3.0; модуль ранжирования реализует описанный алгоритм скоринга; модуль истории хранит контекст сессий в SQLite для поддержки многошагового диалога. Клиентская часть — одностраничное приложение на React (TypeScript). Обмен данными осуществляется по протоколу WebSocket с потоковой передачей статусов обработки, REST API используется как резервный канал. Конвейер обработки запроса включает последовательные этапы: детекция режима → извлечение параметров → поиск на маркетплейсе → ранжирование → выдача топ-5 результатов пользователю.

Разработан и протестирован рабочий прототип системы [4]. Агент корректно классифицирует режим работы, преобразует более 15 абстрактных характеристик звучания в технические параметры и возвращает ранжированные результаты поиска. Обеспечена отказоустойчивость: при недоступности внешних сервисов система переключается на локальные данные с сохранением функциональности фильтрации и ранжирования. Покрытие тестами включает модульные, интеграционные и сквозные сценарии (более 20 тестовых модулей).

К ограничениям текущей версии относятся: поддержка единственного маркетплейса (Reverb); конечный набор маппингов абстрактных характеристик; зависимость качества извлечения параметров от возможностей используемой языковой модели; отсутствие механизма персонализации на основе истории предпочтений пользователя. Перспективные направления развития включают расширение на другие торговые площадки, увеличение таблицы маппингов и внедрение пользовательских профилей.
