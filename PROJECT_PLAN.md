# 1. Цель проекта

Разработать законченный IT-продукт в виде ИИ-агента, который:

* принимает абстрактные пожелания пользователя к звуку;
* переводит их в технические параметры;
* вызывает скрипт поиска на Reverb;
* возвращает 3–5 релевантных ссылок;
* умеет консультировать по тематике гитар.

Продукт должен:

* запускаться по инструкции,
* быть воспроизводимым,
* демонстрировать заявленную функциональность,
* соответствовать требованиям преподавателя .

---

# 2. Организация работы (Full-Stack модель)

## Принципы

1. Все участники — Full-Stack.
2. Нет фиксированных ролей (лид, фронт, бек).
3. Каждый:

   * проектирует,
   * пишет backend,
   * участвует во frontend,
   * работает с LLM,
   * тестирует,
   * участвует в интеграции,
   * пишет документацию.

# 4. Git Flow

## Ветки

* `main` — основная ветка
* `feature/<название>` — рабочие ветки участников
* `hotfix/<название>` — срочные исправления

## Правила

1. В main напрямую не пушим.
2. Работа ведётся через feature-ветки.
3. Merge только через Pull Request.
4. Минимум 1 code review перед merge.
5. Без review merge запрещён.

---

# 5. Code Review (обязательно)

Каждый Pull Request должен:

* иметь описание:

  * что сделано,
  * зачем,
  * какие изменения затронуты;
* пройти минимум 1 проверку другим участником;
* не ломать сборку;
* не нарушать архитектуру.

Проверяем:

* читаемость кода;
* соответствие архитектуре;
* отсутствие хардкода;
* корректность работы с LLM;
* обработку ошибок.

# 9. Базовый системный промпт агента

(Отдельный файл: `AGENT_PROMPT.md`)

Этот текст используется как системный контекст для LLM.

---

## AGENT_PROMPT.md

Ты — ИИ-агент по подбору гитар.

Твоя задача:

1. Анализировать текст пользователя.
2. Извлекать:

   * тип инструмента,
   * бюджет,
   * стиль,
   * абстрактные характеристики звука.
3. Переводить абстрактные характеристики в технические параметры.
4. Формировать структурированный JSON для поиска.
5. Не выдумывать данные.
6. Использовать только результаты, полученные от поискового скрипта.
7. Возвращать только 3–5 релевантных ссылок.
8. Если вопрос консультационный — отвечать без вызова поиска.
9. Не обсуждать доставку и оплату.
10. Не придумывать цены или характеристики.

# 10. Пример абстрактных задач (что будем реализовывать)
